이 매뉴얼의 정확한 사본을 만들고 배포할 수 있는 권한이 부여됩니다. 저작권 공지 및 이 권한 공지가 모든 사본에 보존되어야 합니다.

이 매뉴얼의 수정된 버전을 정확한 복사 조건에 따라 복사하고 배포할 수 있는 권한이 부여됩니다. 단, 전체 파생 작업 결과물이 이 권한 공지와 동일한 조건으로 배포되어야 합니다.

이 매뉴얼을 다른 언어로 번역하여 복사 및 배포할 수 있는 권한이 수정된 버전에 대한 위의 조건에 따라 부여됩니다. 단, 이 권한 공지는 한국R사용자회가 승인한 번역으로 명시될 수 있습니다.

1. 소개

R 재단이 발행하고, 관리하는 R Data Import/Export(번역: R 데이터 가져오기/내보내기) 매뉴얼이 있습니다. 이 기술 문서는 한글로 번역되어 있습니다. 이 매뉴얼, R 데이터 가져오기/내보내기 [한글 부록]은 제목에서 말하는 것처럼 한글 사용자를 위한 R 데이터 가져오기/내보내기의 부록입니다.

이 매뉴얼은 ’한글 관련 추가 설명 제공’이라는 제한된 목적으로 기획되었으며, 이 목적에 부합하도록 최대한 간략하게 작성되었습니다. R 데이터 가져오기/내보내기의 구성 방식과 목차 형식을 따르면서, 한글 사용자가 부딪힐 수 있는 추가 상황에 대응하는 설명을 부분별로 추가하는 것으로 문서 내용이 제공됩니다.

따라서, 이 매뉴얼은 자체적으로는 독립적이거나 완성적인 설명서가 아님을 밝힙니다.

1.1.1 인코딩

맥OS 또는 유닉스 계열의 운영체제를 사용하는 한글 사용자는 이 운영체제들이 utf-8을 기본 인코딩으로 사용한다는 이야기를 들어보았을 것입니다.

하지만, 마이크로소프트사의 윈도우 운영체제를 사용하는 대다수의 사용자들은 한글 윈도우가 어떤 인코딩을 기본으로 사용하는지 모르는 경우가 많습니다. 한글 윈도우는 cp949를 기본 인코딩으로 사용합니다.

일반적으로 R로 자료를 가져올 때, 그리고 내보낼 때, 인코딩은 매우 예민하게 영향을 미치며, 때로는 R 초급자에게는 큰 부담이 되기도 합니다. 다른 운영체제에서 제작된 파일을 가져오거나, 또 다른 운영체제를 사용하는 사용자에게 파일로 데이터를 내보낼 때를 가정해 봅시다. 가져오기/내보내기 관련 함수 안의 인자에서 fileEncoding 또는 encoding 등의 사용 가능 여부를 확인하고, utf-8 또는 CP949, cp949 등의 규칙을 갖는 인코딩 관련 작업을 추가하는 경우가 많기 때문입니다.

예를 들어, 앞서 소개한 scan() 함수의 도움말을 살펴보십시오. fileEncoding, encoding 등의 인자 설명이 있을 것입니다.

1.2 텍스트 파일로 내보내기

R 개발 코어 팀은 R 매뉴얼 홈페이지(https://cran.r-project.org/manuals.html)에서 다음과 같이 밝히고 있습니다:

The following manuals for R were created on Debian Linux and may differ from the manuals for Mac or Windows on platform-specific pages, but most parts will be identical for all platforms.(번역: 다음 R 매뉴얼은 Debian Linux에서 생성되었으며, 플랫폼별 페이지에서 Mac이나 Windows용 매뉴얼과 다를 수 있지만, 대부분의 부분은 모든 플랫폼에서 동일할 것입니다.)

한국알사용자회도 이 문서, R 데이터 가져오기/내보내기 [한글부록]를 Debian와 Debian 계열 리눅스 기반 위에서 제작합니다. 이것을 밝히는 것은 다른 한편으로, 이 문서의 기본 인코딩을 utf-8로 정하고 사용자들에게 한글 도움말을 제공하는 것을 의미합니다. 예를 들어, 윈도우 운영체제에서 제작된 데이터 파일을 불러오는 경우, 또는 윈도우 사용자에게 전달할 목적으로 파일 내보내기를 고려하는 경우 등이 주의를 요구하는 설명 영역이 된다는 뜻이기도 합니다.

write.table() 함수의 도움말에서

 
write.csv(x, file = "foo.csv", fileEncoding = "UTF-16LE")

을 찾을 수 있을 것입니다. 만약 한글 윈도우 사용자에게 파일을 전달할 목적으로

 
write.csv(x, file = "foo.csv", fileEncoding = "CP949")

로 fileEncoding 인자를 수정할 수 있습니다. 주의할 점이 있습니다. cp-949 또는 CP-949 등 중간에 ’-’에 있는 경우, 데이터 내보내기가 성공적이지 않습니다.

2.4 scan 직접 사용

 
> ?scan

을 통하여 scan() 함수의 도움말을 열어서 살펴볼 수 있습니다.

 
scan(file = "", what = double(), nmax = -1, n = -1, sep = "",
          quote = if(identical(sep, "\n")) "" else "'\"", dec = ".",
          skip = 0, nlines = 0, na.strings = "NA",
          flush = FALSE, fill = FALSE, strip.white = FALSE,
          quiet = FALSE, blank.lines.skip = TRUE, multi.line = TRUE,
          comment.char = "", allowEscapes = FALSE,
          fileEncoding = "", encoding = "unknown", text, skipNul = FALSE)

위와 같은 scan() 함수 사용법에서 fileEncoding과 encoding이 있습니다.

fileEncoding: character string: if non-empty declares the encoding used on a file (not a connection nor the keyboard) so the character data can be re-encoded. See the ‘Encoding’ section of the help for ‘file’, and the ‘R Data Import/Export Manual’.(번역: 문자열: 비어 있지 않으면 파일(연결이나 키보드가 아닌)에서 사용되는 인코딩을 선언하여 문자 데이터를 다시 인코딩할 수 있습니다. ’file’ 도움말의 ’Encoding’ 섹션과 ’R 데이터 가져오기/내보내기 매뉴얼’을 참조하십시오.)

encoding: encoding to be assumed for input strings. If the value is ‘"latin1"’ or ‘"UTF-8"’ it is used to mark character strings as known to be in Latin-1 or UTF-8: it is not used to re-encode the input (see ‘fileEncoding’). See also ‘Details’.(번역: 입력 문자열에 대해 가정되는 인코딩입니다. 값이 ’latin1’이거나 ’UTF-8’인 경우, Latin-1 또는 UTF-8로 알려진 문자열을 표시하는 데 사용됩니다. 입력을 다시 인코딩하는 데 사용되지는 않습니다(’fileEncoding’ 참조). ’Details’ 항목도 참조하십시오. )

예를 들어, 만약에 가져올 데이터가 한글 윈도우에서 제작되었다면, fileEncoding 값에 CP949를 우선 고려할 수 있습니다.

3.1 EpiInfo, Minitab, S-PLUS, SAS, SPSS, Stata, Systat

haven 패키지에 대한 작은 설명을 하려고 합니다. haven에는 read_sav()함수가 있으며, foreign의 read.spss()함수와 유사하게 SPSS의 .sav 파일을 가져올 수 있습니다. read_sav()에는 encoding 인자가 포함되어 있어서 다른 인코딩으로 제작된 데이터 파일을 다룰 수 있는 기회를 제공합니다.

 
> library(haven)
> ?read_sav

read_sav() 함수의 도움말 문서를 열고, 함수 사용법을 찾아볼 수 있습니다. 아래와 같습니다:

 
read_sav(
       file,
       encoding = NULL,
       user_na = FALSE,
       col_select = NULL,
       skip = 0,
       n_max = Inf,
       .name_repair = "unique"
)

아래는 read_sav()의 도움말에 있는 encoding에 관한 설명입니다:

encoding: The character encoding used for the file. The default, ‘NULL’, use the encoding specified in the file, but sometimes this value is incorrect and it is useful to be able to override it. (번역: 파일에 사용된 문자 인코딩입니다. 기본값인 ’NULL’은 파일에서 지정된 인코딩을 사용하지만, 때때로 이 값이 잘못되어 있어 재정의할 수 있는 것이 유용합니다.

주의할 점은, Posit사의 RStudio에서 제공하는 외부 데이터 불러오기에 관한 메뉴형 선택기능에 read_sav() 함수가 있는데, encoding 인자가 누락되어 있다는 점입니다. 한글 .sav 파일을 가져오는데 돌발적인 어려움에 부딪힐 가능성이 있습니다. 물론 명령행 입력 방식으로는 성공적으로 처리할 수 있습니다.
